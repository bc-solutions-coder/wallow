## Documentation Rules

- **`docs/` is for the docs site only** — user-facing guides organized by category (getting-started, architecture, development, operations, integrations, api)
- **Do NOT put plans, designs, specs, or session artifacts in `docs/`** — use `docs/plans/` only if explicitly creating a design doc that will become a guide
- **File naming:** lowercase kebab-case (e.g., `api-development.md`), never SCREAMING_CASE
- **New guides** must be added to `docs/toc.yml` under the appropriate category
